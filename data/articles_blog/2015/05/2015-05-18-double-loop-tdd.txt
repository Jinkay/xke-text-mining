2015-05-18-double-loop-tdd|Craft|  Dans ce screencast, Pierre-Jean et moi réalisons ensemble un kata pour démontrer le Double Loop TDD. En résumé, le Double Loop TDD consiste à ajouter un test d’acceptance qui échoue (un test BDD généralement), puis ensuite implémenter la fonctionnalité ajoutée à travers une ou plusieurs boucles TDD. Dans cet exemple, il s’agit d’une application de gestion de comptes bancaires développée en Java. Nous utilisons ici le framework cucumber pour implémenter les tests BDD.    Si vous voulez faire le Kata vous-même, les sources se trouvent sur github.
